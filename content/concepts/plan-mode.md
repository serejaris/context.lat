## Определение

Режим планирования (Plan Mode) — способ взаимодействия с AI-агентом, при котором агент сначала описывает, что собирается сделать (какие файлы создать, какие изменить, в каком порядке), и только после одобрения плана приступает к реализации.

> Планирование занимает 80% времени, реализация — 20%, и это даёт 100% результат.

## Зачем нужен

План можно прочитать, отредактировать, скорректировать до начала работы. Это предотвращает ситуации, когда агент «уходит не туда» и делает ненужные изменения. Особенно полезно для:

- Добавление нового модуля
- Рефакторинг нескольких файлов
- Интеграция с внешним сервисом
- Деплой на новую платформу

## Как использовать

### Вариант 1: Промпт

Добавить в конец промпта:

```
Добавь экран онбординга в моё приложение:
1. Welcome-экран с заголовком и кнопкой "Начать"
2. 3 экрана квиза с вопросами
3. Экран результата

Сначала составь план — какие файлы создать, какие изменить.
Не пиши код, пока я не одобрю план.
```

### Вариант 2: Встроенный режим

| Инструмент | Как включить |
|------------|-------------|
| **Claude Code** | Переключить в plan mode в интерфейсе |
| **Cursor** | Composer → режим Plan |
| **Codex** | По умолчанию планирует перед выполнением |
| **GitHub Copilot** | Agent mode с planning |

## Когда использовать vs не использовать

| Использовать | Не использовать |
|-------------|-----------------|
| Сложная задача, много файлов | Простая правка одного файла |
| Незнакомый проект | Уже знаешь, что нужно сделать |
| Критичные изменения | Прототип / эксперимент |
| Интеграция с внешним API | Стилевые правки |

## Связка с PRD

```
PRD (что строим) → Plan Mode (как строим) → Выполнение
```

На Уроке 2 студенты сначала создавали [PRD](/concepts/prd/) через метапромптинг, затем передавали его [Codex](/tools/codex/) — и Codex автоматически планировал перед выполнением. Два уровня планирования: стратегический (PRD) и тактический (Plan Mode).
