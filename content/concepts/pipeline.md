## Определение

Pipeline — последовательность автоматизированных шагов, где выход одного этапа становится входом следующего. В вайбкодинге типичный pipeline: ресёрч -> черновик -> редактирование -> публикация.

## Пример: контент-фабрика

```
Агент ищет топ-новости за неделю
    ↓
Генерирует посты в авторском стиле
    ↓
Раскладывает в очередь по файлам
    ↓
Планировщик (LaunchD / Task Scheduler) публикует каждый день
```

Каждый шаг можно оптимизировать отдельно: поиск через [Exa MCP](/mcp/exa/), стиль через [системный промпт](/concepts/skill/) с примерами, публикацию через Telegram Bot API.

## Принцип

Всегда начинайте с простого. Самый плохой вариант — потом улучшайте. Первый pipeline может быть одним промптом в Claude Code. Когда процесс устоится — сохраните как скилл и автоматизируйте [планировщиком](/concepts/hooks/).
