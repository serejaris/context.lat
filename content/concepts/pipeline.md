## Определение

Pipeline — последовательность автоматизированных шагов, где выход одного этапа становится входом следующего. В вайбкодинге типичный pipeline: ресёрч → черновик → редактирование → публикация. Каждый шаг можно оптимизировать отдельно, заменять инструменты, масштабировать.

## Пример: контент-фабрика (Урок 4)

```
Агент ищет топ-новости за неделю (EXA)
    ↓
Генерирует посты в авторском стиле (Skill)
    ↓
Раскладывает в очередь по файлам
    ↓
Планировщик (LaunchD / Task Scheduler) публикует каждый день
```

Каждый шаг можно оптимизировать отдельно:
- Поиск — через [Exa MCP](/mcp/exa/)
- Стиль — через [скилл](/concepts/skill/) с примерами авторского голоса
- Публикация — через Telegram Bot API

## Пример: автопубликация в Telegram (Урок 2)

```
Промпт с темой канала
    ↓
Агент генерирует 7 постов на неделю
    ↓
Посты раскладываются в папки по дням
    ↓
Скрипт публикует каждый день в 8:00
```

На Уроке 2 студенты строили этот пайплайн в Cursor: агент получал топ-новости, генерировал посты, раскладывал в очередь. LaunchD запускал публикацию по расписанию.

## Принципы

- **Начинайте с простого** — первый pipeline может быть одним промптом
- **Самый плохой вариант — потом улучшай** — итерируйте каждый шаг отдельно
- **Один скилл = один шаг** — когда процесс устоится, сохраните как [скилл](/concepts/skill/)
- **Автоматизируйте последними** — сначала ручной запуск, потом планировщик

## Pipeline vs одноразовый промпт

| | Одноразовый промпт | Pipeline |
|---|---|---|
| Повторяемость | Каждый раз заново | Один раз настроил — запускаешь |
| Качество | Зависит от промпта | Каждый шаг оптимизирован |
| Масштаб | Одна задача | Серия задач по расписанию |
| Отладка | Непонятно, где ошибка | Каждый шаг отлаживается отдельно |
