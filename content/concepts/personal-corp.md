## Определение

Личная корпорация (Personal Corporation) — концепция организации работы, при которой один человек выступает CEO, а роли отделов ([оркестрация](/concepts/orchestration/)) выполняют AI-агенты. Каждый агент получает системный промпт с описанием своей роли, инструменты и задачи через GitHub Issues.

Сэм Альтман прогнозировал первую компанию-единорог с одним сотрудником. Дарио Амодеи оценил вероятность этого в 2026 году на уровне 70-80%.

## Архитектура

```
CEO (человек)
    |
    |-- Редакция (агент) -- контент, публикации
    |-- Исследования (агент) -- анализ, ресёрч
    |-- Разработка (агент) -- код, деплой
    |-- Маркетинг (агент) -- воронки, рассылки
    +-- Стратегия (агент) -- планирование, OKR
```

Каждый «отдел»:
- Получает системный промпт с описанием роли
- Имеет доступ к нужным инструментам через [MCP](/concepts/mcp/)
- Работает через [GitHub Issues](/tools/github/) как общую память
- Использует [скиллы](/concepts/skill/) как стандартные процедуры

## Пример: редакционный отдел

Первый «отдел», который строят студенты на курсе:

1. Агент-ресёрчер на Haiku собирает топ-темы через [Exa](/mcp/exa/)
2. Агент-редактор на Opus генерирует посты в стиле автора
3. Планировщик (cron + API) публикует в Telegram-канал по расписанию

Результат: ежедневный контент без ручного участия. Стоимость: $0.50-1/день на API.

## Путь масштабирования

```
Один чат-бот --> Агент с инструментами --> Несколько агентов --> Личная корпорация
```

На первом уроке студенты начинают с одного чат-бота. К четвёртому уроку — архитектура из нескольких агентов с разными ролями. Личная корпорация — следующий уровень, где эта архитектура становится основой бизнеса.
