## Определение

Агентский цикл (Agentic Loop) — принцип работы AI-агента: модель получает задачу, генерирует действие (пишет файл, вызывает API, запускает команду), получает результат и решает, продолжать ли работу или задача выполнена. В отличие от чат-бота, который отвечает один раз и ждёт, агент продолжает работу автономно.

## Как работает

```
  Задача
    ↓
  Думает → что нужно сделать?
    ↓
  Действует → вызывает инструмент
    ↓
  Наблюдает → смотрит результат
    ↓
  Готово? — нет → обратно к «Думает»
         — да → ответ
```

Это паттерн **ReAct** (Reason + Act): модель рассуждает о том, что делать, действует, рассуждает о том, что наблюдает, и так по кругу.

## Практические ограничения

Качественно агент работает 1-2 часа, после чего [контекст переполняется](/concepts/context/) и агент теряет фокус. Для длительных задач лучше разбивать работу на сессии с фиксацией прогресса (GitHub Issues, [CLAUDE.md](/concepts/claude-md/)).

## Примеры

- **Claude Code**: получает задачу "добавь авторизацию" → читает проект → планирует → пишет код → запускает тесты → фиксит баги → отчитывается
- **Codex**: получает PR description → анализирует кодовую базу → создаёт ветку → пишет реализацию → открывает Pull Request
