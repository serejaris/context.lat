## Определение

Оркестрация — архитектурный паттерн, при котором одна модель (оркестратор) планирует задачи и делегирует их специализированным агентам ([субагентам](/concepts/subagent/)). Каждый агент получает свой контекст и набор инструментов, работает изолированно и возвращает результат оркестратору. Это позволяет решать сложные задачи параллельно и оптимизировать стоимость: дорогие модели управляют, дешёвые — исполняют.

## Путь масштабирования

```
Один чат-бот (ChatGPT, Claude)
    |
Агент с инструментами (Claude Code)
    |
Агент + субагенты (параллельная работа)
    |
Оркестрация (несколько специализированных агентов)
    |
Личная корпорация (рой агентов как отделы)
```

## Паттерны оркестрации

### Orchestrator-Worker

Основной агент планирует, субагенты исполняют:

```
Opus (оркестратор)
  |-- Haiku: собирает темы новостей
  |-- Haiku: генерирует посты
  |-- Планировщик: публикует по расписанию
```

### [Консилиум агентов](/concepts/agent-council/)

Несколько субагентов запускаются параллельно, каждый с уникальным системным промптом (CMO, CTO, Product Lead). Анализируют один проект с разных сторон → action plan из лучших идей.

### [Pipeline](/concepts/pipeline/)

Цепочка этапов, где результат одного становится входом для следующего:

```
Ресёрч (Exa) → Черновик (Haiku) → Редактура (Opus) → Публикация (API)
```

## Практическое применение

- **Контент-фабрика**: субагент на Haiku собирает темы → основной агент на Opus генерирует финальные посты
- **Code review**: один агент пишет код, другой ревьюит
- **Research**: параллельный поиск по разным источникам через субагентов
- **Стратегия**: консилиум агентов анализирует проект с разных позиций

## Оптимизация стоимости

| Роль | Модель | Стоимость |
|------|--------|-----------|
| Оркестратор | Opus ($15/1M) | Планирует, мало [токенов](/concepts/token/) |
| Исполнитель | Haiku ($0.25/1M) | Много рутинной работы, дёшево |

Opus планирует за $1-2, Haiku исполняет за $0.10-0.20. Общая стоимость на порядок ниже, чем если бы Opus делал всё сам.

## В курсе

| Урок | Контекст |
|------|----------|
| Урок 1 | Введена как тема 2026 года: Claude как планировщик, дешёвые модели как исполнители |
| Урок 2 | Персональная корпорация: архитектор (планирует) + разработчик (реализует) — два агента |
| Урок 3 | Консилиум агентов: CMO, CTO, Product Lead обсуждают стратегию, задачи в GitHub Projects |
| Урок 4 | Субагенты: изолированные сессии. Haiku для черновой работы, Opus для планирования |
