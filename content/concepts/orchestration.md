## Определение

Оркестрация — архитектурный паттерн, при котором одна модель (оркестратор) планирует задачи и делегирует их специализированным агентам ([субагентам](/concepts/subagent/)). Каждый агент получает свой контекст и набор инструментов, работает изолированно и возвращает результат оркестратору. Это позволяет решать сложные задачи параллельно и оптимизировать стоимость: дорогие модели управляют, дешёвые — исполняют.

## Путь масштабирования

```
Один чат-бот
    ↓
Агент с инструментами (Claude Code)
    ↓
Агент + субагенты (параллельная работа)
    ↓
Оркестрация (несколько специализированных агентов)
    ↓
Личная корпорация (рой агентов как отделы)
```

## Паттерн: [Консилиум агентов](/concepts/agent-council/)

Несколько субагентов запускаются параллельно, каждый с уникальным системным промптом (CMO, CTO, Product Lead). Каждый анализирует проект со своей оптикой и выдаёт независимые рекомендации. Финальный результат — action plan из лучших идей.

## Практическое применение

- **Контент-фабрика**: субагент на Haiku собирает темы новостей → основной агент на Opus генерирует финальные посты
- **Code review**: один агент пишет код, другой — ревьюит
- **Research**: параллельный поиск по разным источникам через субагентов
