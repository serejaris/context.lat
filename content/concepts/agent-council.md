## Определение

Консилиум агентов — паттерн, при котором несколько [субагентов](/concepts/subagent/) запускаются параллельно, каждый с уникальным системным промптом в виде персоны. Каждый агент анализирует один и тот же проект со своей оптикой и выдаёт независимые рекомендации. Финальный результат — action plan из лучших идей, собранных из всех точек зрения.

## Как работает

1. Определяются персоны — роли (CMO, CTO, Product Lead) или конкретные люди (Джобс, Дуров)
2. Каждая персона запускается как [субагент](/concepts/subagent/) с уникальным системным промптом
3. Все субагенты получают одинаковый контекст проекта
4. Каждый анализирует со своей оптики
5. Результаты собираются в единый action plan

Не нужно «обучать» агента книжками — Claude Opus уже знает все публичные персоны и роли.

## Пример: консилиум на Уроке 3

На уроке 3 консилиум агентов анализировал реальный проект студента прямо в прямом эфире:

```
CMO: "Нужен лендинг с социальным доказательством.
      Добавить testimonials от первых пользователей."

CTO: "Архитектура слишком сложная для MVP.
      Убрать микросервисы, начать с монолита."

Product Lead: "Метрика успеха не определена.
              Добавить отслеживание retention Day 1/7/30."
```

Рекомендации каждой роли фиксировались как задачи в GitHub Projects.

## Пример: персоны-люди

На менторинге запускался скилл «совет директоров» с реальными персонами: Джобс, Дуров, Айв, Рубчинский — каждый анализировал проект @collage со своей позиции. Результат — 5 конкретных рекомендаций из разных дисциплин.

## Когда использовать

- **Стратегия продукта** — CMO, CTO, Product Lead обсуждают приоритеты
- **Анализ проекта** — разные персоны видят разные проблемы
- **Критика идеи** — несколько точек зрения вместо confirmation bias одного агента
- **Ретро** — несколько перспектив на прошедший спринт

## Как запустить

Консилиум можно запустить как [скилл](/concepts/skill/) через Fetch из GitHub — не нужно писать промпты вручную:

```
→ "Запусти совет директоров для проекта @myapp"
→ Агент: скачивает скилл → запускает 3-5 субагентов
→ Каждый анализирует со своей позиции → собирает action plan
```

Для кастомных персон достаточно описать роль в системном промпте субагента.

## Когда НЕ нужен

- Для простых задач: написать функцию, исправить баг — достаточно одного агента
- При ограниченном бюджете: каждый субагент тратит [токены](/concepts/token/)
- При сжатых сроках: консилиум занимает 5-10 минут
