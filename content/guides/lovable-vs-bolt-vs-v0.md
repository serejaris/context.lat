## Зачем сравнивать

Браузерные AI-билдеры — самый быстрый путь от идеи до работающего продукта. Ничего не устанавливать: описал задачу — получил приложение с публичной ссылкой. Три главных платформы решают разные задачи: визуальный контроль, full-stack прототипы, UI-компоненты.

## Сравнение

| Критерий | [Lovable](/tools/lovable/) | [Bolt.new](/tools/bolt/) | [v0.dev](/tools/v0/) |
|----------|---------|----------|--------|
| Провайдер | Lovable | StackBlitz | Vercel |
| Тип | Full-stack платформа | Full-stack платформа | UI генератор + full-stack |
| Free tier | Да | Да | Да (7 сообщений/день) |
| Рабочий план | Starter $20/мес | Pro $20/мес | Premium $20/мес |
| Топ-план | Launch $50/мес | Team $30/мес | — |
| Визуальный редактор | Да (click-to-edit) | Нет (только чат) | Нет (только чат) |
| Стек | React + Supabase | Любой (Stackblitz WebContainer) | React + Tailwind + shadcn/ui |
| Бэкенд | Через [Supabase](/tools/supabase/) | Встроенный + Supabase, Netlify | Через [Supabase](/tools/supabase/) |
| Деплой | Собственный хостинг + кастомный домен | Netlify (1 клик) | Vercel (1 клик) |
| GitHub sync | Да | Экспорт кода | Нет |
| Кастомный домен | Да | Через Netlify | Только Premium |
| Сила | Визуальный контроль + бэкенд | Мульти-стек, полный проект в браузере | Красивый UI, компоненты |
| Для кого | MVP с визуальным редактированием | Быстрый прототип без установки | UI-компоненты для React |

## Вывод

**[Lovable](/tools/lovable/)** — полноценные приложения с визуальным редактором. Кликай по элементам и меняй — не только через чат. Тесная интеграция с Supabase для базы данных и авторизации. GitHub sync позволяет доработать код в [Cursor](/tools/cursor/).

**[Bolt.new](/tools/bolt/)** — самый быстрый прототип без установки. Всё работает в WebContainer прямо в браузере. Поддерживает любой стек (не только React). Видно, как код создаётся в реальном времени.

**[v0.dev](/tools/v0/)** — красивые UI-компоненты и быстрые веб-приложения на React + Tailwind + shadcn/ui. Три минуты от промпта до публичной ссылки на Vercel. Функция «Enhance prompt» улучшает промпт перед генерацией. Кнопка «Починить» исправляет ошибки без промпта.

**Общий паттерн**: браузерный билдер создаёт прототип, затем код дорабатывается в [Cursor](/tools/cursor/) или [Claude Code](/tools/claude-code/). Билдеры отлично работают для MVP и валидации идеи, но для продакшена обычно нужен переход в полноценную IDE.
